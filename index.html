Why Different Number Systems?

Binary (Base 2): Computers fundamentally operate using binary because digital circuits can represent two distinct states: ON (1) and OFF (0). This simplicity makes it ideal for electronic implementation.

Hexadecimal (Base 16): Hexadecimal is a compact shorthand for binary. Each hex digit (0-9, A-F) precisely represents 4 binary digits (bits). This makes long binary strings much easier for humans to read and write, especially in contexts like memory addresses or color codes.

The CPU and memory are the core components of any computer system, working in tandem to execute programs and process data.

What is a CPU?

The Central Processing Unit (CPU) is often called the "brain" of the computer. It's responsible for executing instructions that make up computer programs, performing calculations, and managing data flow. This is done through a continuous fetch-decode-execute cycle.

üñ±Ô∏è Operating Systems

An Operating System (OS) is software that manages computer hardware and software resources and provides common services for computer programs.

What is an Operating System?

An Operating System (OS) acts as an intermediary between the user and the computer hardware. Its primary goals are to make the computer system convenient to use and to manage hardware resources efficiently.

The Kernel

The kernel is the core component of the operating system. It has complete control over everything in the system. It manages:

  • Process Management: Creating, scheduling, and terminating processes.
  • Memory Management: Allocating and deallocating memory space to processes.
  • File System: Organizing and managing files and directories.
  • Device Drivers: Interacting with hardware devices.
  • Security: Protecting system resources and user data.

Processes

A process is an instance of a computer program that is being executed. It's a fundamental concept in operating systems, each with its own memory space, resources, and execution context.

System calls provide an interface between a process and the operating system. They are the only way a user program can access the kernel's privileged functions.

üíª Command Line Tools

The Command Line Interface (CLI) is a powerful text-based interface for interacting with your computer. It's essential for developers and system administrators.

Essential Commands

Learn some of the most fundamental commands and how they can be chained together using pipes (`|`).

Basic Pipeline Example

A pipeline allows you to combine multiple commands, where the output of one command becomes the input of the next. 

cat file.txt | head -10 | tail -5

This pipeline:
cat file.txt reads the entire content of `file.txt`.
| head -10 takes the output of `cat` and passes only the first 10 lines.
| tail -5 takes those 10 lines and outputs only the last 5 of them.

Common Text Processing Tools

grep and sed are incredibly versatile tools for searching and editing text from the command line.

grep - Search Text

grep "error" logfile.txt    # Finds lines containing "error"
grep -i "ERROR" logfile.txt # Case-insensitive search
grep -n "pattern" file.txt  # Shows line numbers

sed - Stream Editor

sed 's/old/new/' file.txt   # Replaces first "old" with "new" on each line
sed 's/old/new/g' file.txt  # Replaces ALL "old" with "new" on each line
sed '/pattern/d' file.txt   # Deletes lines containing "pattern"

Advanced Pipeline: Word Count

This complex pipeline demonstrates how to find the 10 most common words in a text file. 

cat book.txt | tr ' ' '\n' | sort | uniq -c | sort -nr | head -10

üü¢ Node.js CLI Tools

Node.js is a popular runtime for building fast and scalable network applications, and it's also excellent for creating command-line interface (CLI) tools.

Why Node.js for CLI?

  • JavaScript Everywhere: Use your existing JavaScript knowledge for both frontend and backend/CLI.
  • Huge npm Ecosystem: Access to millions of packages on npm for various functionalities.
  • Cross-Platform: Node.js CLIs run seamlessly on Windows, macOS, and Linux.
  • Great for File Operations: Node.js's built-in fs module makes file system interactions straightforward.
  • Async/Await Support: Handle asynchronous tasks (like network requests or file I/O) elegantly.

Basic CLI Structure (File Line Counter)

This example shows a simple Node.js script that counts the number of lines in a specified file. 

#!/usr/bin/env node

const fs = require('fs'); // Import the file system module

// Get command-line arguments, excluding 'node' and the script name
const args = process.argv.slice(2);

// Check if a filename was provided
if (args.length === 0) {
  console.log('Usage: node script.js <filename>');
  process.exit(1); // Exit with an error code
}

const filename = args[0]; // The first argument is the filename

try {
  // Read the file synchronously (for simplicity in this example)
  const content = fs.readFileSync(filename, 'utf8');
  // Split content by newline and count the resulting array length
  console.log(`Lines: ${content.split('\n').length}`);
} catch (error) {
  console.error(`Error reading file: ${error.message}`);
  process.exit(1);
}

To run this: Save as `line-counter.js`, then `node line-counter.js your_file.txt`

Using npm Packages (Commander & Chalk)

For more robust CLIs, you'll often use npm packages. commander helps parse arguments and commands, while chalk adds styling to your console output. 

// First, install packages: npm install commander chalk

const { Command } = require('commander'); // CLI framework
const chalk = require('chalk');           // For colorful console output

const program = new Command(); // Create a new command instance

// Define basic CLI information
program
  .name('my-cli')
  .version('1.0.0')
  .description('My awesome CLI tool built with Node.js');

// Define a 'hello' command
program
  .command('hello')
  .argument('<name>', 'your name to greet') // Define a required argument
  .action((name) => { // Action to perform when 'hello' command is run
    console.log(chalk.blue(`Hello ${name}! Welcome to the Node.js CLI!`));
  });

// Parse the command-line arguments
program.parse(process.argv);

To run this: Save as `my-cli.js`, then `node my-cli.js hello John`

Basic CLI with argparse (File Analyzer)

argparse is Python's standard library module for parsing command-line arguments. This example creates a simple file analyzer. 

#!/usr/bin/env python3

import argparse # Module for parsing command-line arguments
import sys      # Provides access to system-specific parameters and functions

def main():
    # Create the parser
    parser = argparse.ArgumentParser(
        description='A simple file analyzer CLI tool.'
    )

    # Add positional argument: filename
    parser.add_argument(
        'filename',
        help='The path to the file you want to analyze.'
    )

    # Add optional argument: --lines or -l
    parser.add_argument(
        '--lines', '-l',
        action='store_true', # Stores True if the argument is present
        help='Count and display the number of lines in the file.'
    )

    # Parse the arguments from sys.argv
    args = parser.parse_args()

    try:
        with open(args.filename, 'r') as f:
            lines = f.readlines() # Read all lines into a list
            if args.lines:
                print(f"File: {args.filename}")
                print(f"Total Lines: {len(lines)}")
    except FileNotFoundError:
        print(f"Error: The file '{args.filename}' was not found.")
        sys.exit(1) # Exit with an error code
    except Exception as e:
        print(f"An unexpected error occurred: {e}")
        sys.exit(1)

if __name__ == '__main__':
    main() # Call the main function when the script is executed

To run this: Save as `file_analyzer.py`, then `python3 file_analyzer.py my_document.txt --lines`

Advanced CLI with Click (and Rich for styling)

Click is a popular third-party package for creating beautiful and composable command-line interfaces. We'll also use rich for enhanced console output. 

# First, install packages: pip install click rich

import click
from rich.console import Console # For rich, colorful console output

console = Console() # Initialize Rich console

@click.group() # Decorator to create a group of commands
def cli():
    """My awesome CLI tool built with Python Click."""
    pass # This function acts as the entry point for the CLI group

@cli.command() # Decorator to define a command within the 'cli' group
@click.argument('name') # Define a required positional argument
def hello(name):
    """Say hello to someone with a personalized message."""
    console.print(f"Hello [bold blue]{name}[/bold blue]! Glad you're here.", style="bold green")

@cli.command() # Another command within the 'cli' group
@click.option(
    '--count',
    default=1,
    help='Number of greetings to repeat.',
    type=click.INT # Ensure count is an integer
)
def repeat(count):
    """Repeat a greeting multiple times."""
    if count <= 0:
        console.print("[red]Error:[/red] Count must be a positive number.", style="bold")
        return

    for i in range(count):
        console.print(f"Hello {i+1}!", style="italic yellow")

if __name__ == '__main__':
    cli() # Call the main CLI group when the script is executed

To run this: Save as `my_cli.py`, then `python3 my_cli.py hello Alice` or `python3 my_cli.py repeat --count 3`
